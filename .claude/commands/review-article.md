# /review-article — 기사 교열·팩트체크·SEO 에이전트

발행 전 기사의 언어 교열, 사실 확인 포인트 식별, SEO 최적화를 수행합니다.

## 사용법

```
/review-article
```

검수할 기사 원문을 붙여넣거나 기사 URL/ID를 제공하세요.

## 검수 절차

### 1. 언어 교열

**문법·맞춤법**
- 한국어 어문 규정 준수 확인
- 자주 틀리는 표현:
  - 되다/돼다 → 되다 (기본형), 돼 (해, 해라 대체 가능한 경우)
  - ~로써/로서 → 수단(써), 자격(서)
  - 이/가, 은/는 조사 정확성
- 외래어·전문용어 한국어 표기 확인

**문체 일관성**
- 기사체 (경어 불사용) 유지
- 수동태 → 능동태 전환 제안
- 과도한 피동형 (`~되어지다`) 수정

**가독성**
- 문장 40자 초과 시 분리 제안
- 단락 구성 적절성 (3-4문장 권장)
- 중복 표현 제거

### 2. 팩트체크 포인트 식별

**AI가 직접 확인할 수 없는 항목 → 기자에게 플래그**
- [ ] 인용문 출처 명시됐는지 ("A씨에 따르면" 등)
- [ ] 통계/수치에 출처 표기됐는지
- [ ] 날짜, 장소, 고유명사 정확성
- [ ] 법인명, 직책, 인물명 오기 여부
- [ ] 사건·사고 관련 수사 단계 표현 (피의자/피고인 구분)

**주의 표현 플래그**
- "~로 알려졌다" → 출처 확인 요청
- "~할 것으로 보인다" → 전망 근거 요청
- 범죄 피의자 단정 표현 → 무죄 추정 원칙 적용

### 3. 보도 윤리 검토

- [ ] 취재원/피해자 신원 노출 불필요한 경우 삭제
- [ ] 미성년자 신원 보호 (이니셜 또는 익명 처리)
- [ ] 자살/자해 관련 보도 지침 준수 (한국자살예방협회 기준)
- [ ] 광고성 표현 기사에 포함됐는지 (네이티브 광고 명시 필요)

### 4. SEO 최적화

**제목 분석**
- 현재 제목: [기사 제목]
- 길이: N자 (35자 이하 권장)
- 핵심 키워드 포함 여부
- 개선 제안: [제안 제목 3개]

**메타 설명**
- 현재: [메타 설명]
- 길이: N자 (120-150자 권장)
- 클릭 유도 문구 포함 여부
- 개선 제안

**URL 슬러그**
- 현재: /news/[slug]
- 권장: 핵심 키워드 포함, 20자 이내, 하이픈 구분

**키워드 밀도**
- 핵심 키워드 자연스럽게 분포됐는지 (과도한 반복 지양)
- 지역명 "울산" 포함 여부 (로컬 SEO)

## 주의사항
- 이 에이전트는 언어적 검토와 체크리스트를 제공하며, 실제 사실 확인은 기자가 직접 수행
- 법적 검토 필요 항목(명예훼손, 저작권 등)은 법무팀에 별도 의뢰
